Pseudo-Randomness

While we can have sources of entropy, in some cases, it is more convenient to generate pseudo-random numbers. Unlike pure random numbers, these are generated from a purely random initial number (seed) to obtain a large enough sequence of unpredictable numbers. Through certain algorithms, sequences generated from a seed are chaotic enough to give the impression of being random.

This process, following an algorithm, is deterministic, meaning: for the same seed, the sequence generated will be the same. A characteristic of pseudo-random number generators (PRNG) is the period, i.e., for how long a pseudo-random sequence can be generated without returning to the same initial point, i.e., the seed. This means that in certain PRNG algorithms, the chain can repeat indefinitely reached a length. This directly depends on the length of the seed: the longer the seed, the longer the period will be.

It is desirable that a PRNG satisfies the following properties:

  • Uniformity: Generated numbers should follow a uniform distribution. Therefore, each number in the specified range has the same probability of being generated.
  • Independence: The generation of a number should not depend on previously generated numbers. There should also be no pattern with which, given the previous numbers, the next one can be predicted. We can visually verify in any Random Walker if this property is satisfied by observing that no predictable patterns occur.
  • Long Periodicity: The generator should have a long enough period before it starts repeating the sequence of generated numbers. A short period could result in the same sequence repeating over and over, being completely predictable. We can visually verify in any Random Walker if this property is satisfied by observing that the drawn path does not repeat.
  • Reproducibility: When starting the generator with a seed, it should always generate the same frequency for that same seed. This is also useful for reproducing experiments with the same set of input data or for video games. We can visually check this property in the Random Walker with custom seed input.
  • Efficiency: It should be efficient in runtime and resource usage, generating numbers quickly without requiring too many computational resources.
